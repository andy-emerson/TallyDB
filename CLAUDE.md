# Start here

This file is a pointer, kept thin so agent instructions and project design
stay separate. Read, in order:

1. **`AGENTS.md`** — how we work in this repository: roles, passes, reviews,
   the claim-grading system, wants vs. decisions, and how work reaches
   `main`.
2. **`DESIGN.md`** — what we are building and why: positioning, the three
   assumptions, invariants, crate boundaries, settled decisions, and build
   order.

`README.md` is the user-facing view of the project's current state. Open
work and decisions live in GitHub Issues and Milestones, not in files.

Do not add project guidance to this file; it stays a pointer.
